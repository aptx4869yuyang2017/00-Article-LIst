> 网上的 Markdown 教程数不胜数，我想写一篇有点不一样的“指南”。

Markdown 的语法内容真的是太简单了，基本上只要一个小时，基本上所有的语法规则都能过一遍，随便套用几个语法，修改一下别人设定好的 CSS 渲染的模板，基本上就能产出一篇，排版精美，方便传播的文档。
真的不想在重复那些语法内容了，以下列出最常用的语法规则。
* 标题
* 加粗
* 斜体
* 块引用
* 连接
* 图片
* 列表
* 分割线
